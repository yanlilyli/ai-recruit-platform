⚪ L0 - 无需具备
N/A / 不作要求
定义：该能力维度对本岗位非核心要求，候选人无需具备亦可通过。但若具备，可作为加分参考。
使用场景
• 该岗位核心是别的方向，AI能力非主要考核
• 用于显式标注"不要求"，避免面试官误判
• 仅作为锦上添花的能力背景了解
🟢 L1 - 了解
Familiar / 应用调用级
定义：理解基本概念和原理，能调用现成API/SDK完成基础任务，能跑通开源Demo或公开教程。
表现示例（AI算法方向）
• 能调用大模型API（OpenAI/Claude）并写出基础Prompt
• 了解Transformer、Attention等基本算法概念
• 能跑通开源SFT/LoRA训练脚本
• 了解强化学习基本概念（Reward / Policy）
适用场景
应用开发岗的基础门槛 / 跨职能岗（PM、运营）的加分项
🔵 L2 - 熟练
Proficient / 业务落地级
定义：具备扎实的工程化能力，能独立完成数据准备、模型微调或工作流搭建，能将AI技术与游戏业务深度结合并解决常见问题。
表现示例（AI算法方向）
• 独立完成开源大模型的SFT/LoRA微调（含数据准备）
• 熟练搭建复杂Agent工作流（多工具调用、规划与反思）
• 能基于RLHF/DPO做模型对齐与偏好优化
• 使用PPO/MARL训练可上线的游戏AI Bot
适用场景
AI应用开发主力 / 后训练工程师 / 能独立负责一个AI特性落地
🟣 L3 - 精通
Expert / 算法与架构级
定义：深入理解底层算法与架构原理，具备核心问题攻坚能力。能改进或重构模型结构，或在特定场景下提出创新的AI解决方案。能发一作顶会论文是综合能力的集中体现。
表现示例（AI算法方向）
• 在NeurIPS/ICML/ICLR/CVPR等顶会发表一作论文
• 能基于业务需求改进或重构模型网络结构
• 精通预训练Scaling Law、分布式训练优化
• 能从零设计复杂RL环境与奖励函数并解决收敛问题
• 推理性能极致优化（量化/算子/调度）
适用场景
AI算法研究员 / 后训练核心专家 / AI技术架构师

🔄 派单流程

1. 需求方下单：明确岗位的AI能力要求（必备/加分） + 最低L级
2. 管理员（你+总监+组长）评定：在面试官池中维护每位面试官的能力评定
3. 系统智能推荐：派单时按"必备项L级覆盖度"排序面试官
4. 派单：你勾选合适的面试官（支持多人交叉面试），系统记录派单结果
5. 面试结果回填：面试完成后管理员标记完成状态
