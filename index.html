📐 AI能力掌握程度 · 统一标尺

为了拉通全组评估标准，避免"精通"通货膨胀，所有需求方填写「最低要求」、所有面试官评定「可面层级」均按此标尺执行。
⚪ L0 - 无需具备
N/A / 不作要求
定义：该能力维度对本岗位非核心要求，候选人无需具备亦可通过。但若具备，可作为加分参考。
使用场景
• 该岗位核心是别的方向，AI能力非主要考核
• 用于显式标注"不要求"，避免面试官误判
• 仅作为锦上添花的能力背景了解
🟢 L1 - 了解
Familiar / 应用调用级
定义：理解基本概念和原理，能跑通开源Demo，能使用现成的工具或API完成基础任务。
表现示例
• 能调用大模型API并编写基础Prompt
• 能按教程部署并运行Stable Diffusion
• 了解强化学习基本概念
• 能使用现成的插件库
适用场景
业务侧需要简单接入AI能力，或作为复合型人才的加分项
🔵 L2 - 熟练
Proficient / 业务落地级
定义：具备扎实的工程化能力，能将AI技术与游戏业务（UE5/服务器）深度结合，能独立完成数据准备、模型微调或工作流搭建，并解决常见Bug。
表现示例
• 熟练使用LangChain等框架搭建复杂Agent工作流
• 独立完成开源大模型/CV模型的微调（SFT/LoRA）
• 能将ONNX等推理引擎接入游戏客户端
• 使用PPO等算法训练游戏AI Bot
适用场景
AI应用开发的主力军，能独立负责一个AI特性的落地
🟣 L3 - 精通
Expert / 算法与架构级
定义：深入理解底层算法与架构原理，具备核心问题攻坚能力。能解决极端的性能瓶颈（算力、内存、延迟），或针对特定游戏场景设计创新的AI解决方案。
表现示例
• 能基于业务需求改进或重构模型网络结构
• 精通推理加速与底层优化（TensorRT、CUDA算子）
• 能从零设计复杂RL环境与奖励函数
• 具备前沿Paper的复现与工程化落地能力
适用场景
解决核心技术难题的AI专家、底层架构师或算法研究员

🔄 派单流程

1. 需求方下单：明确岗位的AI能力要求（必备/加分） + 最低L级
2. 管理员（你+总监+组长）评定：在面试官池中维护每位面试官的能力评定
3. 系统智能推荐：派单时按"必备项L级覆盖度"排序面试官
4. 派单：你勾选合适的面试官（支持多人交叉面试），系统记录派单结果
5. 面试结果回填：面试完成后管理员标记完成状态
